TareaFarmacia.java

La clase TareaFarmacia representa una tarea dentro de la farmacia, almacenando información relevante como un identificador único, descripción, tipo (urgente, receta, inventario, administrativa o prioritaria), prioridad (alta, media o baja), fecha de creación, estado (pendiente, en proceso o completada) y el tiempo estimado para realizarla.

// Ejemplo de creación de una tarea
TareaFarmacia tarea = new TareaFarmacia(
  1,
  "Reabastecer medicamentos en vitrina",
  TipoTarea.INVENTARIO,
  Prioridad.MEDIA,
  30 // minutos estimados
);

El constructor inicializa todos estos atributos y establece el estado inicial como PENDIENTE. Los métodos getter permiten acceder a los atributos, mientras que el setter modifica el estado de la tarea. El método estaCompletada verifica si el estado de la tarea es COMPLETADA. El método toString ofrece una representación textual de la tarea, mostrando sus datos principales de forma legible.

Consejo: Utilice tipos de tarea específicos para facilitar la búsqueda y filtrado posterior.

Esta clase facilita la creación, consulta y gestión del estado de las tareas en el sistema de farmacia.

Node.java

La clase Node es el elemento básico para construir estructuras de datos enlazadas, como listas, pilas y colas, dentro del sistema de farmacia.

Estructura de un Nodo:

[Nodo Anterior] ← [Nodo Actual: TareaFarmacia] → [Nodo Siguiente]

Cada nodo almacena un objeto de tipo TareaFarmacia y una referencia al siguiente nodo en la estructura. El constructor inicializa el nodo con una tarea y la referencia al siguiente nodo como nula. Los métodos getter y setter permiten acceder y modificar tanto la tarea como el siguiente nodo.

// Ejemplo de creación de nodos enlazados
Node primerNodo = new Node(tarea1);
Node segundoNodo = new Node(tarea2);
primerNodo.setNext(segundoNodo); // Enlazar nodos

Pila.java

La clase Pila representa una pila de tareas urgentes para la farmacia, utilizando nodos enlazados.

Funcionamiento de una Pila (LIFO):

Tarea 3 (Cima - Última en entrar)
Tarea 2
Tarea 1 (Fondo - Primera en entrar)

Permite:

  • Apilar nuevas tareas en la cima.
  • Procesar la tarea más urgente, marcándola como completada.
  • Completar una tarea específica por ID.
  • Consultar la cima sin eliminarla.
  • Verificar si está vacía y contar su tamaño.
  • Mostrar tareas pendientes (ordenadas por prioridad) y completadas.

ListaTareas.java

La clase ListaTareas implementa una lista enlazada para gestionar tareas generales.

Inserción
Agregar tareas al final de la lista
Búsqueda
Encontrar tareas por ID, tipo o descripción
Eliminación
Remover tareas específicas por ID
Estadísticas
Contar tareas pendientes y calcular tiempo total

Incluye funcionalidades como:

  • Insertar tareas al final.
  • Completar o eliminar tareas por ID.
  • Buscar tareas por ID, tipo o descripción.
  • Contar tareas pendientes.
  • Mostrar pendientes y completadas ordenadas por prioridad.
  • Calcular recursivamente el tiempo total de tareas pendientes.
  • Obtener tareas ordenadas por prioridad y fecha.
Consejo: Utilice la búsqueda por descripción cuando no recuerde el ID exacto de una tarea.

Esto permite una administración detallada y flexible de las tareas de la farmacia.

ColaPrioridad.java

La clase ColaPrioridad implementa una cola de prioridad mediante un heap binario.

Comparación de Estructuras:

Estructura Orden de Procesamiento Uso Ideal
Pila (LIFO) Último en entrar, primero en salir Tareas urgentes que requieren atención inmediata
Cola (FIFO) Primero en entrar, primero en salir Tareas en orden de llegada
Cola de Prioridad Por prioridad (no por orden de llegada) Tareas con diferentes niveles de importancia

Permite insertar tareas, consultar la de mayor prioridad, procesarla (marcarla como completada y eliminarla), completar tareas por ID y mostrar todas ordenadas.

La comparación entre tareas se hace primero por estado (pendientes primero), luego por nivel de prioridad y finalmente por fecha de creación (más antiguas primero).

// Ejemplo de uso de Cola de Prioridad
ColaPrioridad cola = new ColaPrioridad();
cola.insertar(tareaAltaPrioridad);
cola.insertar(tareaMediaPrioridad);
TareaFarmacia proximaTarea = cola.consultar(); // Devuelve tareaAltaPrioridad

Empleado.java

La clase Empleado representa a un trabajador de la farmacia, con atributos como ID, nombre, departamento, puesto, habilidades y carga de trabajo.

Calculadora de Disponibilidad:

/10

Incluye métodos para verificar disponibilidad, asignar carga, comprobar habilidades y obtener representación textual del empleado. Facilita la organización y consulta de las capacidades y disponibilidad de los empleados.

Consejo: Asigne tareas considerando tanto las habilidades del empleado como su carga de trabajo actual para optimizar la productividad.

EmployeeNode.java

La clase EmployeeNode representa un nodo en un árbol binario de empleados.

Estructura de un Árbol Binario:

        [Nodo Raíz]
        /        \
[Hijo Izquierdo]  [Hijo Derecho]
    /    \            /    \
[Nieto] [Nieto]   [Nieto] [Nieto]

Cada nodo contiene un empleado y referencias a hijos izquierdo y derecho. Esta clase sirve como base para construir árboles binarios de empleados, facilitando la organización jerárquica y la búsqueda eficiente dentro de la estructura.

EmployeeTree.java

La clase EmployeeTree implementa un árbol binario que organiza empleados por un criterio configurable (ID, nombre o carga de trabajo).

Inserción Balanceada
Mantiene el árbol equilibrado para búsquedas eficientes
Búsqueda Rápida
Encuentra empleados en tiempo logarítmico
Asignación Óptima
Sugiere el empleado más adecuado para cada tarea
Estadísticas
Proporciona datos sobre la distribución de empleados

Permite insertar, buscar, asignar tareas óptimas, consultar empleados por departamento y mostrar estadísticas.

// Ejemplo de búsqueda en el árbol
EmployeeTree arbol = new EmployeeTree(Criterio.CARGA_TRABAJO);
// ... agregar empleados al árbol
Empleado empleadoOptimo = arbol.asignarTareaOptima(tarea);

EmployeeManager.java

La clase EmployeeManager gestiona empleados con árboles binarios y mapas, facilitando búsquedas y asignaciones.

Permite agregar empleados, asignar tareas de forma óptima, mostrar empleados disponibles por habilidades y distribuir cargas de trabajo mediante un algoritmo de divide y vencerás.

Algoritmo Divide y Vencerás: Este enfoque divide el problema en subproblemas más pequeños, los resuelve recursivamente y combina las soluciones. En la gestión de empleados, permite distribuir equitativamente la carga de trabajo.

Main.java

La clase Main centraliza el sistema de gestión de farmacia mediante un menú en consola.

Flujo del Programa Principal:

Inicio → Menú Principal → Opción Seleccionada → Ejecución → Volver al Menú

El menú permite:

  • Agregar, procesar, completar y eliminar tareas.
  • Mostrar estadísticas de tiempo estimado.
  • Agregar empleados, buscarlos, mostrarlos por departamento o disponibilidad.
  • Asignar tareas óptimas a empleados.
// Estructura básica del método main
public static void main(String[] args) {
  boolean salir = false;
  while (!salir) {
    mostrarMenu();
    int opcion = leerOpcion();
    salir = procesarOpcion(opcion);
  }
}

Guía de Uso del Sistema

  1. Iniciar el sistema: Ejecuta el archivo Main.java. Se abrirá un menú interactivo en la consola.
  2. Menú principal: Selecciona la opción deseada escribiendo el número correspondiente.

Opciones comunes:

  • Agregar tarea: introduce los datos solicitados (ID, tipo, descripción, prioridad, tiempo estimado).
  • Procesar tarea: selecciona si deseas procesar una tarea urgente, receta, prioritaria o general.
  • Completar/Eliminar tarea: ingresa el ID de la tarea a gestionar.
  • Mostrar estadísticas: revisa cuántas tareas pendientes existen y el tiempo estimado total.

Gestión de empleados

  • Agregar empleado: introduce ID, nombre, departamento, puesto y habilidades.
  • Mostrar empleados por departamento: consulta la organización jerárquica.
  • Buscar por ID o habilidad: facilita localizar empleados específicos.
  • Asignar tareas: el sistema sugiere automáticamente el empleado más adecuado según carga de trabajo y habilidades.

Consejos prácticos

  • Mantén actualizada la carga de trabajo de los empleados para una asignación óptima.
  • Completa las tareas a tiempo para mantener la lista organizada.
  • Usa las búsquedas por tipo y descripción para encontrar tareas rápidamente.
  • Revisa regularmente las estadísticas para identificar cuellos de botella.

Estructuras de Datos Utilizadas

El sistema utiliza diversas estructuras de datos para optimizar el manejo de tareas y empleados:

Estructura Uso en el Sistema Ventajas
Lista Enlazada Gestión de tareas generales Flexibilidad para insertar y eliminar elementos
Pila Tareas urgentes (LIFO) Acceso rápido al elemento más reciente
Cola de Prioridad Tareas ordenadas por importancia Procesamiento por prioridad, no por orden de llegada
Árbol Binario Organización jerárquica de empleados Búsquedas eficientes (O(log n))
Mapa/HashMap Búsqueda rápida de empleados por ID Acceso constante (O(1)) a los elementos
Nota: La elección de cada estructura de datos está optimizada para el tipo de operación que se realiza con más frecuencia en cada caso.

Escenario 1: Gestión de una receta urgente

  4. La tarea se programa para el momento de menor afluencia de clientes

Flujos de Trabajo

El sistema sigue flujos de trabajo específicos para diferentes tipos de tareas:

Flujo para Tareas Urgentes

Creación de Tarea Urgente → Pila de Urgentes → Asignación Inmediata
↓
Procesamiento → Verificación → Completado

Flujo para Tareas Programadas

Creación de Tarea → Lista General → Asignación por Prioridad
↓
Programación → Ejecución → Completado

Flujo de Gestión de Empleados

Registro de Empleado → Árbol Binario → Asignación de Tareas
↓
Seguimiento de Carga → Reasignación si es necesario
Optimización: Los flujos están diseñados para minimizar el tiempo de respuesta en tareas críticas y maximizar la eficiencia en tareas programadas.
